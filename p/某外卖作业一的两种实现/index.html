某外卖作业一的两种实现

2026-04-02

缓存商户类型shopType的List和String实现方式

整体逻辑：Controller → Service → ServiceImpl → Redis / MySQL
Controller：接请求、返回数据
Service：写接口
ServiceImpl：写具体逻辑（重点）

List 实现方式

Controller：

@RestController
@RequestMapping("/shop-type")
public class ShopTypeController {
    @Resource
    private IShopTypeService shopTypeService;

    @GetMapping("list")
    public List<ShopType>  queryTypeList() {
        return shopTypeService.queryTypeList();
    }
}

很简单，直接调用service层
Service：

public interface IShopTypeService extends IService<ShopType> {

    List<ShopType> queryTypeList();

}

ServiceImpl（List实现核心）：

@Servicehttp://localhost:1313/p/%E6%9F%90%E5%A4%96%E5%8D%96%E4%BD%9C%E4%B8%9A%E4%B8%80%E7%9A%84%E4%B8%A4%E7%A7%8D775140915015
public class ShopTypeServiceImpl extends ServiceImpl<ShopTypeMapper, ShopType> implements IShopTypeService {
    @Resource
    private StringRedisTemplate stringRedisTemplate;


    @Override
    public List<ShopType> queryTypeList() {
        String key=SHOP_TYPE_LIST_CACHE_KEY;
        // 1. 查 Redis（List）
        List<String> list = stringRedisTemplate.opsForList()
                .range(key, 0, -1);

        // 2. 判断是否命中
        if (list != null && !list.isEmpty()) {
            return list.stream()
                    .map(s -> JSONUtil.toBean(s, ShopType.class))
                    .collect(Collectors.toList());
        }

        // 3. 查数据库
        List<ShopType> shopTypes = this.query()
                .orderByAsc("sort")
                .list();

        if (shopTypes == null || shopTypes.isEmpty()) {
            return shopTypes;
        }

        // 4. 写入 Redis（一个一个存）
        for (ShopType shopType : shopTypes) {
            stringRedisTemplate.opsForList().rightPush(
                    SHOP_TYPE_LIST_CACHE_KEY,
                    JSONUtil.toJsonStr(shopType)
            );
        }

        return shopTypes;
    }
}

这里在redisConstants定义了：

public static final String SHOP_TYPE_LIST_CACHE_KEY = "shop:type:list";

类定义：

@Service
public class ShopTypeServiceImpl 
    extends ServiceImpl<ShopTypeMapper, ShopType> 
    implements IShopTypeService

@Service：Spring 管理这个类（自动注入）
extends ServiceImpl：MyBatis-Plus 提供的基础类
让你可以直接用 this.query()
implements IShopTypeService：实现接口

注入 Redis：

@Resource
private StringRedisTemplate stringRedisTemplate;

注入一个操作 Redis 的工具类

定义 key：

String key = SHOP_TYPE_LIST_CACHE_KEY;

查redis：

List<String> list = stringRedisTemplate.opsForList()
        .range(key, 0, -1);

opsForList()获取 Redis 的 List 操作对象
从 Redis 取数据：.range(key, 0, -1)，相当于读取整个列表
json->对象：

return list.stream()
        .map(s -> JSONUtil.toBean(s, ShopType.class))
        .collect(Collectors.toList());

Stream()是java流式处理
map()是把每个JSON都转化成JSONUtil.toBean(s, ShopType.class)，即ShopType对象
Collect()收集成List，最终得到List 如果redis没数据：

List<ShopType> shopTypes = this.query()
        .orderByAsc("sort")
        .list();

这里是mybatis-plus的写法 ① this.query()

👉 MyBatis-Plus 提供的查询构造器

② .orderByAsc(“sort”)

👉 按字段 sort 升序

③ .list()

👉 执行 SQL

相当于：

SELECT * FROM shop_type ORDER BY sort ASC;

写入redis

for (ShopType shopType : shopTypes) {
    stringRedisTemplate.opsForList().rightPush(
            SHOP_TYPE_LIST_CACHE_KEY,
            JSONUtil.toJsonStr(shopType)
    );
}

① JSONUtil.toJsonStr(shopType)

👉 把对象转成 JSON：

ShopType → “{id:1,name:美食}” ② rightPush()

👉 往 Redis List 右边追加

👉 类似：

list.add() 📦 Redis 最终结构： key: shop:type:list

value: [ “{…}”, “{…}”, “{…}” ] 🟢 🔟 返回结果 return shopTypes;

String 实现方式

Controller，Service一样，service层前面一样，从queryTypeList的实现开始不同

@Override
public List<ShopType> queryTypeList() {
    String key=SHOP_TYPE_LIST_CACHE_KEY;
    // 1. 查 Redis（String）
    String json = stringRedisTemplate.opsForValue().get(key);

    // 2. 命中缓存
    if (StrUtil.isNotBlank(json)) {
        return JSONUtil.toList(json, ShopType.class);
    }

    // 3. 查数据库
    List<ShopType> shopTypes = this.query()
            .orderByAsc("sort")
            .list();

    if (shopTypes == null || shopTypes.isEmpty()) {
        return shopTypes;
    }

    // 4. 写入 Redis（一次写）
    stringRedisTemplate.opsForValue().set(
            key,
            JSONUtil.toJsonStr(shopTypes)
    );

    return shopTypes;
}

个人感觉string实现更加明晰点

维度 List String
存储方式 多条 一条
IO次数 多 少
代码复杂度 高 低
更新方式 局部可改 必须整体
推荐程度 ⭐⭐ ⭐⭐⭐⭐
